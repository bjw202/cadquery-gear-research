# Project Operating Rules

## Agent policy

- 기본적으로 가능한 작업은 멀티에이전트로 분해한다.
- 메인 에이전트는 직접 구현보다 조율, 계획, 사용자 응답을 우선한다.
- 메인 에이전트는 사용자의 새 질문에 즉시 응답 가능 상태를 유지한다.

## Documentation policy

- 모든 중요한 논의는 회의록/개발저널/의사결정 로그에 기록한다.
- 결정 사항은 "배경 / 선택지 / 결정 / 이유 / 후속 액션" 형식으로 남긴다.
- 세션 종료 전 반드시 오늘 변경사항 요약과 다음 시작점을 기록한다.

## Design-first policy

- 구현 전에 문제정의, 대안 비교, 성공조건, 실패조건을 먼저 정리한다.
- 사용자가 요구한 산출물이 실제 문제 해결에 맞는지 먼저 검증한다.

## Output policy

- 보고서는 요약 → 근거 → 대안 → 권고 순으로 작성한다.
- 불확실한 내용은 추정이라고 표시한다.
