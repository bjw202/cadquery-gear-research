- **최종 아키텍처**:

  2. **LLM 역할 한정**: 공정 지시서 자연어 생성, 비표준 재질 예외 처리, 사용자 질의 응답만 담당. 절삭 조건 수치 단독 생성 금지.
  3. **RAG 공구 DB 연동**: Sandvik CoroPlus 등 공구 카탈로그 + Machining Handbook 데이터를 JSON으로 구조화. 1단계는 JSON 직접 참조, 이후 벡터 DB로 확장.
  4. **DEC-009 아키텍처 확장**: 기존 `gear_library` + `cadquery-gear` 스킬에 `gear-process-planning` 스킬을 추가하는 방식으로 통합.
- **이유**: 순수 LLM으로 가공 조건 수치를 생성하면 할루시네이션 위험(항공우주 사례 근거). 규칙 엔진은 입증된 정확도, RAG는 최신 공구 데이터 반영, LLM은 문서 품질 향상에 집중.
- **최종 통합 아키텍처**:
  ```
  사용자 입력 (기어 파라미터 + 재질 + 정밀도 등급 + 수량)
      ↓
  [모델링 스킬] generate_gear.py → STEP/STL 출력
      ↓
  [공정 스킬] gear_process_engine.py + data/*.json
      → 가공 방법 선택 (규칙 기반)
      → 공구 규격 매핑 (JSON DB)
  ```
